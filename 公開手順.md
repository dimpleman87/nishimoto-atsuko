# 西本敦子 ホームページ 無料公開手順

## 推奨URL名

公開名は `nishimoto-atsuko` を推奨します。

例：

- GitHubリポジトリ名：`nishimoto-atsuko`
- Netlifyサイト名：`nishimoto-atsuko`
- URL例：`https://nishimoto-atsuko.netlify.app`

「西本敦子」の日本語をURLに入れることもできますが、共有時に長い英数字へ変換される場合があります。見た目の分かりやすさでは、ローマ字の `nishimoto-atsuko` が実務上安全です。

## ファイル構成

この `site_nishimoto` フォルダをそのまま公開できます。

- `index.html`：ホームページ本体
- `styles.css`：旧CSS。現在の主要デザインは `index.html` 内に入っています。
- `assets/`：写真、チラシ、公式LINE QR

## 無料公開のおすすめ

### 1. Netlify Drop

一番簡単です。相談フォームもNetlify Formsとして動かせるため、初期公開はこちらを推奨します。

1. `site_nishimoto` フォルダをZIP化する
2. Netlify Drop にアクセスする
3. ZIPまたはフォルダをドラッグ＆ドロップする
4. 発行されたURLを確認する
5. Site settings からサイト名を `nishimoto-atsuko` に変更する

URL：
https://app.netlify.com/drop

### 2. GitHub Pages

今後もGitで更新するならこちらが安定です。ただし、GitHub Pages単体では相談フォーム送信は処理できません。

1. GitHubで新しいリポジトリを作る
   - Repository name：`nishimoto-atsuko`
2. `site_nishimoto` の中身をアップロードする
3. Settings → Pages を開く
4. Branch を `main`、Folder を `/root` にして公開する

URL例：
`https://GitHubユーザー名.github.io/nishimoto-atsuko/`

### 3. Cloudflare Pages

独自ドメインを使う予定がある場合に向いています。

1. GitHubにファイルを置く
2. Cloudflare Pages でGitHubリポジトリを接続する
3. Framework preset は `None`
4. Build command は空欄
5. Output directory は空欄または `/`

## 公開前チェック

- 公式LINEボタンが開くか
- QRコードがスマホで読み取れるか
- 電話リンクがスマホで発信画面になるか
- メールリンクがメール作成画面になるか
- YouTubeミニ画面が表示されるか
- 相談フォームを使う場合、Netlifyの管理画面でFormsに送信が届くか

## 小規模事業者向けの運用注意

- Netlify、GitHub、Google、LINE公式アカウントは二段階認証を有効にする
- ログイン情報をメールやチャットにそのまま貼らない
- 相談フォームで受け取った個人情報は、必要な担当者だけが見られる場所で管理する
- LINE公式アカウントの管理者権限は必要最小限にする
